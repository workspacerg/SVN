Utilisation avec tortoise


Maintenant que notre repo est prêt, nous allons commencer à voir les bases de son utilisation avec tortoise SVN.

Tortoise est un programme qui rajoute dans votre menu contextuel des options liées à SVN.

IMG

Créer par exemple un dossier nommé svnRepo sur le même ordinateur ou un sur le même réseau local. Il servira à utiliser nos dépôts.

En effet, n'oublions pas que SVN se sert d'un dépot centralisé, de ce fait, le repo sur le serveur contient les données et les envoie sur les autres comptes utilisateurs ayant les droits d'accès lors d'un pull (récupération de données). Il ne faut jamais modifier directement le dépôt coté serveur.

Ensuite, créer un dossier pour notre projet "monTest".

  • Effectuer un click droit pour faire apparaître le menu deroulant.

  • Ensuite, depuis SVN, cliquer sur Checkout (permet de recuperer le projet depuis le serveur).

Deux cas s'offrent à vous. Dans l'adresse vous avez besoin d'entrer le lien vers votre serveur :

  • https://localhost/svn/

  • https://NomDuPcServer/svn/

Enfin, a droite, cliquer sur le bouton comportant 3 points. Un login et un mot de pase vous seront demandé.

Cliquer sur trunk puis valider

Voila, vous pouvez commencer à travailler sur votre projet.

Commit

Nous verrons plus en details par la suite les possibilités de SVN et du versionning en général, mais je vais quand même faire un petit tour rapide pour commencer.

Un commit correspond à l'envoi de modifications sur le serveur.

Exemple :

Créer par exemple un fichier index.html, ensuite pour qu'il soit pris en compte, vous devez l'ajouter à l'index de svn en cliquant droit -> tortoise -> Add . Une croix bleu apparait sur le dossier et un point d'exclamation rouge sur l'icone du dossier.

Dernière étape, cliquer droit -> commit ...

Si tout ce passe bien un tick vert apparait et votre fichier est sur le serveur. Si vous ne me faites pas confiance, allez sur visual server puis actualiser la page et vous verrez que votre fichier est apparu dans le dossier trunck.

Update

Avant chaque modification, et dans l'idéal le plus souvvent possible, il vous faudra faire des updates, c'est à dire récupérer les modifications des autres personnes sur le repo (sauf si vous êtes seul bien sur).

De la meme manière clique droit -> svn Update tout simplement.
