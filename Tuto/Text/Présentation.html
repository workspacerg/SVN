Utilisation avec tortoise


Maintenant que notre repo est près, nous allons commencer à voir les bases de son utilisation avec tortoise SVN.

Tortoise est un programme qui rajoute dans votre menu contextuel des options lié à SVN.

IMG

Crée par exemple un dossier nommée svnRepo sur le même ordinateur ou un sur le même réseaux local. Il servira à utilisé nos dépôts.

En effet, n'oublions pas que svn est ce sert d'un dépot centralisé donc le repo sur le serveur contient les données et les envoie sur les autres utilisateur aillant les droits d'accès lors d'un pull. Il ne faut jamais modifié directement dans le dépôt coté serveur.

Ensuite, crée un dossier pour notre projet "monTest".

  • Cliquez droit pour faire apparaitre le menu deroulant.

  • Ensuite sur svn Chekout ( permet de recuperer le projet depuis le serveur).

Deux cas soffre à vous dans l'adresse vous avez besoin detaper le lien vers votre serveur soit en :

  • https://localhost/svn/

  • https://NomDuPcServer/svn/

Enfin a droite cliquez sur le bouton au trois points, un login et un mot de pase vous sera demandé.

cliquez sur trunk et valider

Voila vous pouvez commencer à travailler sur votre projet.

Commit

Nous verrons plus en details par la suite les possibilités de SVN et du versionning en général, mais je vais quand même faire un petit tour rapide pour commencer.

Un commit correspond à l'envoie de modifications sur le serveur.

Exemple :

Crée par exemple un fichier index.html, ensuite pour qu'il soit pris en compte, vous devez l'ajouter à l'index de svn en cliquand droit -> tortoise -> Add . Une croix bleu apparait sur le dossier et un point d'explamation rouge sur l'icone du dossier.

Dernière étapes, clique droit -> commit ...

Si tout ce passe bien un tick vert apparait et votre fichier est sur le server. Si vous ne me faite pas confiance, allez sur visual server actualiser la page et vous verrez que votre fichier est apparu dans le dossier trunck.

update

Avant chaque modification et dans l'idéale le plus souvvent possible, il vous faudra faire des updates, c'est à dire récupérer les modifications des autres personnes sur le repo (sauf si vous êtes seul bien sur ).

De la meme manière clique droit -> svn Update tout simplement.
